Browser TypeScript → HTTP → Python proxy → NATS → Go Telepact server

Telepact full-stack proxy demo

The browser sends Telepact request bytes over HTTP to a Python proxy. The proxy does not use Telepact at all; it only forwards the raw bytes to the NATS subject encoded in the request URL. A Go Telepact server listens on that subject and replies with Telepact response bytes.

Route over HTTP

proxy: loading POST /rpc/rpc.demo.greet outcome: none

What this example demonstrates

  • the browser can use plain HTTP while the internal RPC transport stays on NATS
  • the proxy forwards raw request and response bytes without importing Telepact
  • the NATS subject is chosen from the HTTP URL instead of hard-coded proxy logic
  • the Go server still speaks normal Telepact once the bytes arrive on its topic
