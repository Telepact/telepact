Python backend + websocket proxy + TypeScript browser frontend

Telepact full-stack websocket proxy demo

This browser app still uses the dashboard schema and browser UI from the full-stack example, but sends Telepact request bytes to a websocket proxy. The proxy forwards the raw bytes to NATS, and the real Telepact server handles auth normalization, metrics, request IDs, and generic unknown-error responses behind that transport boundary.

What this example demonstrates

  • the browser talks to a websocket proxy instead of directly to the Telepact server
  • the proxy forwards raw request bytes to a NATS topic without any Telepact code
  • on_auth still normalizes identity into internal Telepact headers
  • request IDs, per-function metrics, and structured events still come from Telepact hooks
  • ErrorUnauthorized_ stays close to the admin-only business rule
  • ErrorUnknown_ still keeps the wire response generic while exposing a caseId
