Python backend + TypeScript browser frontend

Telepact full-stack production-boundary demo

This browser app talks to a Python Telepact server over HTTP. The server extracts a session cookie into +auth_, normalizes identity with on_auth, emits Telepact-aware metrics and logs, attaches request IDs, keeps auth decisions near the handlers, and surfaces unexpected failures as ErrorUnknown_ with a caseId.

What this example demonstrates

  • transport-specific cookie extraction stays at the HTTP edge
  • on_auth normalizes identity into internal Telepact headers
  • request IDs, per-function metrics, and structured events come from Telepact hooks
  • ErrorUnauthorized_ stays close to the admin-only business rule
  • ErrorUnknown_ still keeps the wire response generic while exposing a caseId
  • schema compatibility is tracked with the checked-in schema-baseline/ snapshot
