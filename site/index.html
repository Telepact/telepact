Three Key Innovations

Each innovation targets a different axis of API design, creating an ecosystem that is simultaneously accessible, efficient, and expressive.

{ }

JSON as a Query Language

API calls and SELECT-style queries are all achieved with JSON abstractions, giving first-class status to clients wielding only a JSON library.

No query params. No string parsing. Just JSON in, JSON out.
01

Binary Without Code Generation

Binary protocols are established through runtime handshakes, rather than build-time code generation, offering binary efficiency without code generation toolchains.

No field IDs in your schema. No ABI leaking into your API.
→

Hypermedia Without HTTP

API calls can return functions with pre-filled arguments, approximating a link that can be followed, achieved with pure JSON abstractions over any transport.

Navigate your API like a graph. Transport-agnostic links.

It's Just JSON

No query params. No binary field IDs. No required client libraries. It's just JSON in, and JSON out.

api/math.telepact.yaml 
- ///: Divide two integers, `x` and `y`.
  fn.divide:
    x: "integer"
    y: "integer"
  ->:
    - Ok_:
        result: "number"
    - ErrorCannotDivideByZero: {}
Define your API with a clean, YAML-based schema. Functions have typed arguments and tagged result unions.
server.py 
from telepact import TelepactSchemaFiles, TelepactSchema, Server, Message

async def divide(function_name, request_message):
    arguments = request_message.body[function_name]
    x = arguments['x']
    y = arguments['y']
    if y == 0:
        return Message({}, {'ErrorCannotDivideByZero': {}})
    result = x / y
    return Message({}, {'Ok_': {'result': result}})

schema_files = TelepactSchemaFiles('./api')
api = TelepactSchema.from_file_json_map(schema_files.filenames_to_json)
server = Server(api, {'fn.divide': divide}, options)
Serve it using one of the Telepact libraries (Python, TypeScript, Java, or Go) over any transport.
client.js 
// No Telepact library needed — just fetch + JSON
let request = [{}, {"fn.divide": { x: 6, y: 3 }}];

let response = await fetch(
    "http://localhost:8000/api/telepact",
    {
        method: "POST",
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(request),
    },
);

console.log(await response.json());
// [{}, {"Ok_": {"result": 2}}]
Call it from any language with nothing but a JSON library and an HTTP client. No Telepact dependency needed.

See It on the Wire

Every Telepact message is a two-element JSON array: headers and body. Click through a real conversation.

→ Request Simple function call 
[{}, {"fn.divide": {"x": 6, "y": 3}}]
← Response Tagged result union 
[{}, {"Ok_": {"result": 2}}]
→ Request Error case — divide by zero 
[{}, {"fn.divide": {"x": 6, "y": 0}}]
← Response Typed error — no status codes, no guessing 
[{}, {"ErrorCannotDivideByZero": {}}]
→ Request Select only specific fields from response 
[{"@select_": {"->": {"Ok_": ["result"]}}},
 {"fn.evaluate": {"expression": {"Add": ...}}}]
← Response Only selected fields returned — like GraphQL, but JSON 
[{}, {"Ok_": {"result": 42}}]

Standout Features

Explore the capabilities that set Telepact apart.

🎯 Dynamic Response Shaping

Like GraphQL's field selection, but with pure JSON. Use the @select_ header to trim response fields — no special query language needed.

Select only what you need 
// Request: only return "result" from Ok_
[
  {"@select_": {
    "->": {"Ok_": ["result"]}
  }},
  {"fn.evaluate": {
    "expression": {"Add": {
      "left": {"Constant": {"value": 1}},
      "right": {"Variable": {"name": "x"}}
    }}
  }}
]

// Response: only selected fields
[{}, {"Ok_": {"result": 42}}]

🔗 Hypermedia Links

Responses can include pre-populated function calls as typed links. Clients can follow them to navigate the API, just like clicking a link on a webpage.

API links in responses 
// Response includes a link to save the result
[{}, {"Ok_": {
  "result": 10,
  "saveResult": {
    "fn.saveVariable": {
      "name": "result",
      "value": 10
    }
  }
}}]

// Client can "follow" the link by sending it as a request
[{}, {"fn.saveVariable": {
  "name": "result",
  "value": 10
}}]

⚡ Binary Without Field IDs

Telepact negotiates binary encoding at runtime through a client-server handshake. No field IDs in your schema, no ABI drift, no generated code required.

Automatic binary negotiation 
// Client opts in with a single header
[{"@bin_": {}}, {"fn.divide": {"x": 6, "y": 3}}]

// Telepact automatically negotiates MessagePack encoding
// Subsequent messages are compact binary
// No schema field IDs. No code generation. Just smaller payloads.

// The server handles it transparently:
response = await server.process(request_bytes)
// ^ works for both JSON and binary requests

🧪 Built-in Mock Server

Test against a full mock server with programmable stubs and call verification — all part of the Telepact protocol. No extra test frameworks needed.

Mock server with stubs 
// Create a stub
[{}, {"fn.createStub_": {"stub": {
  "fn.getUser": {"id": "user-1"},
  "->": {"Ok_": {"user": {
    "id": "user-1", "name": "Ada"
  }}}
}}}]

// Verify a call was made
[{}, {"fn.verify_": {"call": {
  "fn.getUser": {"id": "user-1"}
}}}]
// -> [{}, {"Ok_": {}}] ✓ verified!

Design Principles

🌐

Accessibility

Whether you bring sophisticated toolchains or a minimalist setup, you can participate in Telepact with the complexity level you need.

📦

Portability

Telepact definitions are currency in the ecosystem, unlocking documentation, code completion, mocking, and code generation.

🔒

Trust

Features are governed by the ecosystem itself, so clients can confidently integrate with all Telepact servers with robust expectations.

⚓

Stability

A carefully curated list of paradigms ensures API designs don't fall victim to common API evolution pitfalls.
