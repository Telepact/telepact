Three innovations that redefine APIs

The Telepact protocol fuses the best of JSON, binary, and hypermedia to eliminate integration friction between services and clients.

JSON as a query language

Telepact treats JSON as a first-class query mechanism. Clients express projections, filters, and selections with the tooling they already understand.

Binary without build chains

Runtime handshakes negotiate binary encodings so you can tune bandwidth and latency without locking into heavy code generation workflows.

Hypermedia beyond HTTP

Return callable functions with pre-filled arguments to guide consumers through flows—no HTTP required, just portable JSON payloads.

Get started in minutes

Use your preferred language to serve and consume Telepact APIs.

Define your API

Author a schema file that captures the functions and shapes you want to expose.

[
    {
        "fn.divide": { "x": "integer", "y": "integer" },
        "->": [ { "Ok_": { "result": "number" } }, { "ErrorCannotDivideByZero": {} } ]
    }
]

Schemas live alongside your codebase in .telepact.json files.

Serve over any transport

Bring your preferred framework. Telepact handles schema negotiation, message parsing, and response encoding.

  • Drop-in server adapters for HTTP, WebSockets, queues, and more
  • Runtime binary handshakes for efficient payloads
  • Auth strategies with shared headers and envelopes
